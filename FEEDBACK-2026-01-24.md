#### 3. Script Language Mix is Incorrect
**Problem:** The dialogue mixes English and Italian within phrases incorrectly. They speak half-English, half-Italian in the same sentence.

**Current Behavior:** Appears to be code-switching inappropriately (e.g., "Io want to mangiare la pizza")

**Correct Michel Thomas Format:**
- **Teacher explanations:** Pure English ("Now, in Italian, the word for 'to eat' is...")
- **Italian demonstrations:** Pure Italian ("Io mangio la pizza")
- **Translation requests:** English prompt ("How would you say 'I eat pizza'?")
- **Student attempts:** Should be Italian (with possible mistakes in Italian grammar/vocab)
- **Corrections:** Italian phrase + English explanation

**Example of CORRECT flow:**
```
TEACHER: In Italian, "to eat" is "mangiare". Can you repeat that?
STUDENT: Mangiare.
TEACHER: Perfect. Now, how would you say "I eat"?
STUDENT: Io mangio.
TEACHER: Excellent! Now try "I eat pizza".
STUDENT: Io mangio pizza.
TEACHER: Almost perfect! In Italian we say "la pizza". So: "Io mangio la pizza".
```

**Example of INCORRECT flow (what we're getting now):**
```
TEACHER: Now we learn mangiare, which is to eat in italiano.
STUDENT: So I say io mangio when I want to eat?
TEACHER: Yes, perfetto! You mangiare la pizza.
```

**Root Cause:** Prompt instruction says "Include both English and Italian naturally in the dialogue (bilingual mix)" which is being misinterpreted as code-switching rather than structured teaching.

**Fix Required:**
- Update prompt to clarify language separation
- Teacher speaks English for instruction, Italian for demonstration
- Student attempts should be in Italian (not mixed)
- No code-switching within sentences

**Next Steps:**
- ✅ Reviewed episode-001-script.txt - see EPISODE-001-ANALYSIS.md for detailed breakdown
- Rewrite prompt to specify clear language boundaries
- Add examples of correct dialogue format to prompt

**Detailed Analysis:** See `EPISODE-001-ANALYSIS.md` for complete breakdown of issues and root cause.

#### 4. Pacing Too Fast - Missing Thinking Gaps
**Problem:** Episode speaks too quickly without pauses for user to think and respond

**Context:** Michel Thomas method effectiveness comes from:
- Teacher asks a question in English
- **PAUSE** - student (listener) thinks about answer
- Student attempts answer
- **PAUSE** - listener compares their answer
- Teacher provides correction/confirmation

**Current State:** Script likely has gaps in dialogue, but TTS is reading them without sufficient pause duration

**Solutions Needed:**

1. **Script Generation Level:**
   - Update prompt to explicitly request pause markers
   - E.g., `[PAUSE 5 seconds]` or `[Wait for student response]`

2. **Audio Generation Level:**
   - Parse pause markers from script
   - Insert actual silence periods in audio
   - Options:
     - Generate segments separately with silence between
     - Use audio library (pydub) to insert silence
     - Use SSML breaks if TTS supports it

3. **Pause Strategy:**
   - After teacher question: 5-8 seconds
   - After student attempts: 3-5 seconds
   - Between vocabulary items: 2-3 seconds

**Example Flow:**
```
TEACHER: Now Alex, how would you say "I eat pizza" in Italian?
[PAUSE 7 seconds]
STUDENT: Um... io... mangio... la pizza?
[PAUSE 4 seconds]
TEACHER: Excellent! Io mangio la pizza. Perfect pronunciation.
```

**Next Steps:**
- Update GPT prompt to include explicit pause markers
- Modify audio generation to handle pauses
- Test different pause durations for optimal learning
